# iCloud Privacy Mail 发布与使用说明

## 1. 这是什么

这是一个 Go 单文件后端项目，前端页面已经通过 `embed` 打进程序里。

解压后可以直接二开，也可以直接编译运行。

## 2. 目录说明

- `cmd/panel/main.go`：程序入口。
- `internal/app/`：后端、Apple/iCloud 协议、数据存储、页面模板。
- `internal/app/templates/`：前端页面模板。
- `config.example.json`：示例配置。
- `README.md`：完整功能说明。
- `iCloud隐私邮箱取码平台接入设计.md`：接口和设计记录。

源码包不包含真实数据：

- 不包含 `data/state.json`
- 不包含真实 `config.json`
- 不包含账号、Cookie、邮箱 API Key
- 不包含 `.git` 历史目录
- 不包含编译产物

## 3. Windows 本地运行

安装 Go 后，在项目目录执行：

```powershell
copy config.example.json config.json
go run ./cmd/panel --config config.json
```

默认打开：

```text
http://127.0.0.1:8787/
```

也可以编译成 exe：

```powershell
go build -trimpath -o icloud-privacy-mail.exe ./cmd/panel
.\icloud-privacy-mail.exe --config config.json
```

## 4. Linux 本地运行

```bash
cp config.example.json config.json
go run ./cmd/panel --config config.json
```

编译 Linux amd64：

```bash
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -trimpath -ldflags="-s -w" -o icloud-privacy-mail ./cmd/panel
chmod +x icloud-privacy-mail
./icloud-privacy-mail --config config.json
```

## 5. 第一次管理员账号

系统没有默认管理员账号密码。

第一次打开页面后，自己注册账号：

- 第一个注册成功的账号会自动成为管理员。
- 后续注册账号是普通用户。

## 6. 配置文件

复制 `config.example.json` 为 `config.json` 后按需修改：

```json
{
  "host": "127.0.0.1",
  "port": 8787,
  "data_path": "data/state.json",
  "api_key": "",
  "public_base_url": "",
  "icloud_default_host": "www.icloud.com.cn",
  "icloud_client_id": "d39ba9916b7251055b22c7f910e2ea796ee65e98b2ddecea8f5dde8d9d1a815d"
}
```

关键字段：

- `host`：监听地址。本地建议 `127.0.0.1`。
- `port`：监听端口，默认 `8787`。
- `data_path`：数据文件路径。真实账号、登录态、邮箱和邮件缓存都在这里。
- `api_key`：全局自动取号 API Key，可留空。
- `public_base_url`：复制邮箱 API 时使用的外部域名，例如 `https://www.example.com`。
- `icloud_default_host`：Apple/iCloud 默认域，国内账号常用 `www.icloud.com.cn`，海外账号可用 `www.icloud.com`。

## 8. systemd 示例

创建 `/etc/systemd/system/icloud-privacy-mail.service`：

```ini
[Unit]
Description=iCloud Privacy Mail
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/icloud-privacy-mail
ExecStart=/opt/icloud-privacy-mail/icloud-privacy-mail --config /opt/icloud-privacy-mail/config.json
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动：

```bash
systemctl daemon-reload
systemctl enable --now icloud-privacy-mail
systemctl status icloud-privacy-mail
```

查看日志：

```bash
journalctl -u icloud-privacy-mail -f
```

## 9. Nginx 反代示例

```nginx
server {
    listen 80;
    server_name 你的域名;

    location / {
        proxy_pass http://127.0.0.1:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

如果使用 HTTPS，请在 Nginx 或 Caddy 层配置证书。

## 10. 升级注意事项

升级程序时只替换二进制，不要覆盖数据文件：

```bash
systemctl stop icloud-privacy-mail
cp icloud-privacy-mail /opt/icloud-privacy-mail/icloud-privacy-mail
systemctl start icloud-privacy-mail
```

升级前建议备份：

```bash
cp /opt/icloud-privacy-mail/data/state.json /opt/icloud-privacy-mail/data/state.json.bak
```

不要把自己的 `state.json` 发给别人。

### 标准升级方式

本项目模板通过 `go:embed` 打进二进制，升级前端和后端都只需要替换 Linux 二进制。

本地构建：

```powershell
$env:CGO_ENABLED='0'
$env:GOOS='linux'
$env:GOARCH='amd64'
go build -trimpath -ldflags "-s -w" -o icloud-privacy-mail ./cmd/panel
```

远端升级原则：

1. 新建 `/opt/icloud-privacy-mail/releases/<release-name>/`。
2. 上传新二进制到该 release 目录并设置 `0755`。
3. 写入 `icloud-privacy-mail.sha256` 并执行 `sha256sum -c`。
4. 记录旧 `current` 指向到 `/opt/icloud-privacy-mail/backups/<release-name>-previous-release.txt`。
5. 原子切换 `/opt/icloud-privacy-mail/current` 软链接。
6. `systemctl restart icloud-privacy-mail.service`。
7. 先测 `http://127.0.0.1:8787/login`，再测公网域名。

不要上传或覆盖：

- `/opt/icloud-privacy-mail/shared/config.json`
- `/opt/icloud-privacy-mail/shared/data/state.json`
- `/opt/icloud-privacy-mail/shared/` 下任何真实运行数据

### 回滚方式

如果新版本服务启动失败或烟测失败，把 `current` 切回上一个 release 后重启：

```bash
old=/opt/icloud-privacy-mail/releases/20260629-ui-categories-pagination-copyapi
ln -sfn "$old" /opt/icloud-privacy-mail/current.next
mv -Tf /opt/icloud-privacy-mail/current.next /opt/icloud-privacy-mail/current
systemctl restart icloud-privacy-mail.service
systemctl status icloud-privacy-mail.service --no-pager
```

如果是后续版本，优先读取对应备份记录：

```bash
cat /opt/icloud-privacy-mail/backups/<release-name>-previous-release.txt
```

## 12. 常用操作

启动后进入首页：

```text
http://127.0.0.1:8787/
```

使用流程：

1. 注册第一个管理员账号。
2. 登录后台。
3. 在页面里保存 Apple/iCloud 登录态。
4. 创建或同步 Hide My Email 隐私邮箱。
5. 复制每个邮箱的 API 地址给其他项目取验证码。

对外取码 API 格式：

```text
http://你的域名/api/v1/mailboxes/邮箱地址/code?key=邮箱独立Key
```

如果短时间重新发送验证码，普通取码接口会跳过已经返回过的同一封旧验证码邮件；需要排查缓存时才加：

```text
&cache=1
```
