# 更新日志

本文只记录公开版代码变更，不包含真实账号、服务器地址、登录态、Cookie、验证码、API 密钥或运行数据。

## 2026-06-30

### 新接口登录态保活与刷新

- 修复 Apple Account 新接口短效管理态被当成长效状态复用的问题。
- 新接口会按 Apple 返回的管理态 TTL 到期时间刷新 `scnt`、Cookie 和动态 `apiKey`，再使用刷新后的状态创建。
- 新接口创建失败时只保存最后一次成功响应得到的刷新态，非 2xx 失败响应不再覆盖 `scnt` 或 Cookie。
- 新接口浏览器指纹调整为抓包一致的中文语言和东八区时区，减少管理态检测与创建接口的指纹漂移。
- 新接口 401/403 不再一律判定为登录态失效；只有明确返回 authentication/session 失效语义时才要求重新登录，普通 301/404/500/503 会保留原始 HTTP 错误。
- 新接口创建与登录态检测按同一个账号串行执行，避免检测刷新 `scnt` 时创建接口同时使用旧状态。
- 新接口达到隐私邮箱创建上限时会进入短冷却，当前轮回落旧接口或跳过该账号，下一轮再重新尝试。
- 登录前获取 Apple Account 挑战 `scnt` 与登录后管理态保存拆开处理，避免挑战响应污染已保存的新接口登录态。
- 新接口 2FA 通过后会尝试补 Apple 信任确认；失败时不阻断正常保存，只在调试模式输出脱敏摘要。
- 明确说明浏览器 “Remember me” 不等同于旧 iCloud Web 长效 Cookie；旧接口长效来自 `2sv/trust` 和 `extended_login` 换取的 iCloud WebServices Cookie。

### 登录态检测与展示

- 登录态检测同时覆盖 Apple Account 新接口和 iCloud Web 旧接口。
- 新接口登录态检测会复用近期健康状态，避免刚保存的新接口状态被检测刷新误判为异常。
- 每个 Apple 登录态会分别展示“新接口”和“旧接口”的登录状态。
- 前端卡片精简为 Apple ID、账号 ID、新旧接口状态和整体登录态结果。
- Apple ID / 账号邮箱在前端全量显示，不再用星号脱敏。
- 去掉前端卡片里多余的最近检测时间、Cookie 数量、DSID、保存时间等调试信息。
- 新增测试覆盖新接口登录态检测、公开状态字段和新旧接口状态拆分。

### 管理面板 UI

- 管理员后台改为新的分类化数据面板，顶部动作区和数据卡片重新整理。
- 首页提供清透、蓝调、灰阶、舒展、紧凑等前端风格切换组件。
- 账号信息与“账号数据/退出”入口移动到右上角，减少左侧栏拥挤。
- 导出和清理工具从左侧栏移动到“高级工具”分类中，不再常驻挤占导航。
- 日志区域统一展示运行日志、创建账号日志和定时创建日志；切换普通分类时日志保留，切到邮箱池时隐藏。

### 隐私邮箱创建

- 隐私邮箱创建优先使用 Apple Account 新接口。
- 新接口创建前会优先使用近期可用的 `scnt` 和动态 `apiKey`，真正认证失败时再刷新并重试。
- 新接口创建失败后，会继续尝试 iCloud Web 旧接口。
- 定时创建按账号并行处理；某个账号本轮失败后，只跳过该账号当前轮次，其他账号继续创建。
- 定时创建只保留定时间隔配置，默认 60 分钟；每轮创建间隔可自定义，默认 5 秒。
- 普通创建和定时创建都支持选择“自动使用新旧接口 / 只用新接口 / 只用旧接口”。
- 创建配置保存到服务器状态文件，刷新页面或换浏览器后仍会保留账号选择、接口策略和定时间隔。
- 定时创建记录已创建数量、失败数量、下一次执行时间和最近错误。
- 定时创建记录支持复制日志和清除日志；清除只清记录，不重置创建/失败计数。
- 创建账号列表里会完整显示 Apple ID，并清楚标记新接口、旧接口和整体登录态状态。
- 创建区域新增“检测登录态”入口，可直接刷新新旧接口状态显示。
- 创建失败会展示账号级失败原因；Apple 新接口错误会带阶段、HTTP 状态码和脱敏后的原始返回摘要，方便判断是接口入口异常、登录态失效、额度上限还是服务端临时错误。
- Apple 新接口 2xx 空响应按临时失败处理，下一轮优先重试新接口，再次失败才回落旧接口继续。
- Apple 新接口遇到 DNS 超时、连接超时、连接重置等底层网络错误时会自动重试 3 次；HTTP 业务错误、额度上限和登录态失效不走网络重试。
- 新接口刷新倒计时到点后会自动调用登录态检测接口，真正刷新 Apple Account 管理态时间，不再只刷新页面状态。
