# 更新日志

本文只记录公开版代码变更，不包含真实账号、服务器地址、登录态、Cookie、验证码、API 密钥或运行数据。

## 2026-07-02

### 发布边界

- 明确公开版发布规则：每次修复功能、问题或公开行为，都必须提升 `AppVersion`、提交 Git，并发布新的 GitHub Release。
- 在线更新只读取最新 Release 或 manifest 最新版本；用户即使跳过中间多个版本，也会直接更新到最新版本，不需要逐个版本升级。
- Release 资产只包含程序二进制和校验文件，不包含 `data/state.json`、配置、登录态、Cookie、验证码缓存或服务器私有数据。

### 在线更新与版本展示

- 前端顶部新增当前版本显示；鼠标悬停可查看最新版本号、发布时间、更新包名称和更新内容。
- 新增 `/api/update/status` 检查更新接口，默认读取公开 GitHub 最新 Release，也支持配置 `update_manifest_url` 使用自定义 manifest。
- 当 GitHub 仓库还没有发布 Release 时，不再把 `releases/latest` 的 404 显示成检查失败；后端会退回读取默认分支最新 commit，用当前二进制内置 commit 判断是否已经是最新源码。
- 管理员可在检测到可用二进制资产后点击“更新”，服务会下载当前系统架构的程序文件并替换当前二进制；`data/state.json` 不参与更新，不覆盖用户数据。
- 更新配置新增 `update_enabled`、`update_repository`、`update_manifest_url`、`update_asset_name`，适合公开版部署者无需重新拉源码即可升级。

### Apple Account 保活展示

- 新接口登录态已检测异常时，前端不再继续显示下一次保活倒计时，改为显示“新接口保活：已停止”。
- 后台新接口保活 worker 会跳过已检测异常的登录态，避免异常状态下继续保活并造成前端误判；手动检测成功或重新登录后才会重新进入保活队列。

## 2026-07-01

### Apple Account 新接口保活

- 根据长时间浏览器 HAR 对比确认：`/v2/jslogs` 会持续返回新的浏览器会话标识，但不能证明隐私邮箱管理接口仍可用；真实入口 `/account/manage/email/private` 和 `/account/manage/forwardemail` 仍可能在页面保持打开时返回 401。
- 后台新接口保活改为随机错峰触发真实管理接口：刷新 `gs/ws/token`、重新读取 `/account/manage` 动态 `apiKey`，再轻量请求 `/account/manage/forwardemail` 验证管理态仍可操作。
- 后台保活 worker 使用短周期扫描，到点的账号才执行保活；不同账号按 owner/account/Apple ID 生成稳定随机偏移，避免所有账号同一秒打到 Apple。
- `/v2/jslogs` 仅作为补充同步浏览器会话头，不再作为新接口登录态正常的唯一依据。
- “检测登录态”现在会触达真实管理接口，避免只刷新时间但点隐私邮箱入口仍掉登录态的误判。
- 前端新接口倒计时显示改为后端下一次保活时间，不再由前端循环自动刷新生成。
- 新增测试覆盖真实管理接口保活请求顺序和后台保活写回已保存登录态。

### Apple 登录验证码方式

- 新接口登录和旧接口登录都支持选择 2FA 验证码方式：受信任设备或短信/手机号验证码。
- 登录验证码方式纳入服务器端创建配置，刷新页面后仍会保留，不再只存在浏览器本地。
- 登录按钮文案从“保存新/旧接口登录态”精简为“新接口登录 / 旧接口登录”。
- 新增测试覆盖验证码方式配置保存，以及新接口短信验证码提交链路。

### 隐私邮箱同步

- “同步已有邮箱”继续固定使用 iCloud Web 旧接口 `/v2/hme/list`，新接口只用于创建邮箱，不参与同步已有邮箱。
- 全部登录态同步改为账号级并发隔离；某个账号卡住或超时只会记录该账号失败，不再拖到整次请求 504。
- 同步结果会记录每个账号实际使用的旧接口和失败原因，避免只显示笼统的“同步失败”。
- 根据 iCloud CN 抓包确认旧接口已有邮箱列表仍为 `GET /v2/hme/list`，并将旧接口默认前端 build 号更新到 `2622Build20`；已保存 build 的登录态不受影响。
- 新增测试覆盖旧接口列表解析和慢账号隔离，确保一个账号超时不影响其他账号同步入库。

### 公开取码 API

- 后台取码同步器接入 IMAP `IDLE` 常驻监听，收到 `EXISTS` 新邮件事件后立即触发同步入库。
- IMAP 同步改为优先使用取码登录态的账号级 `IMAPLastSyncUID` 做 `UID n:*` 增量搜索，兼容旧邮箱级 `LastSyncUID`，没有 UID 游标时才按日期回看兜底。
- IMAP 监听同步不再为没有新验证码的空邮箱逐个写入同步游标，避免大邮箱池反复刷写 `state.json` 堵住公开取码 API。
- 后台取码监听启动时会先读取 iCloud IMAP 当前 `UIDNEXT` 并保存为账号级起跑线，之后只同步新进来的邮件，不再启动就翻历史邮件。
- IDLE 监听 worker 不再因为邮箱同步游标更新而重启，避免刚入库就断开常驻监听。
- 大邮箱池同步前会先构建取码登录态索引，避免每个邮箱重复查找/克隆登录态导致同步上下文被耗尽。
- IMAP 连接改为 TCP 公共 DNS 优先，并以 UDP DNS、本机 resolver 和 Apple IMAP IPv4 直连兜底，降低服务器 DNS/53 端口偶发超时对取码同步的影响；直连仍使用 `imap.mail.me.com` 做 TLS SNI。
- 原 3 秒后台轮询保留为兜底，防止 IMAP 事件漏掉或连接被 Apple 断开。
- 修复公开取码 API 等待补抓时丢失请求 `keyword` 的问题；外部传入 `keyword=ChatGPT` 时，后端会按 ChatGPT 同步验证码邮件，不再固定用 OpenAI。
- 新增面板取码预览模式；后台面板点击“验证码”只用于查看和复制，不会占用公开 API 的已发放验证码记录。
- 修复取码请求等待期间后台监听已把验证码入库、但本次请求没有再次读取本地缓存导致第一次返回空、第二次才秒出的竞态；等待过程中会实时轮询本地入库结果，发现验证码立即返回。
- 面板“验证码”按钮不再使用浏览器本机时间生成 `after` 参数，避免客户端时间偏差或 5 分钟边界把服务器已有验证码过滤掉。
- 新增测试覆盖 ChatGPT keyword 补抓、面板预览不消耗公开 API、IMAP EXISTS 事件识别和 UID 增量搜索。

## 2026-06-30

### 验证码读取提速

- 服务启动后默认启用后台取码同步器，按 Apple 账号批量同步活跃隐私邮箱的验证码邮件。
- 后台取码同步器默认 3 秒一轮，启动时预抓最近 24 小时内的最近 20 个邮件线程，后续每轮扫描最近 8 个线程。
- 对外取码 API 改为本地优先；本地未命中时触发后台快速补抓，默认最多等待 600ms，补抓继续在后台完成。
- 取码同步继续复用原有 iCloud Mail WebService、同步游标、owner 级同步锁和同步最小间隔，避免高频轮询打爆 Apple 接口。
- 新增 `MAIL_WATCHER_*` 和 `PUBLIC_*SYNC*` 配置项，公开说明里补充“后台先入库、查询读本地”的提速模型。
- 新增测试覆盖 watcher 配置读取、启动预抓写入验证码、本地未命中时快速返回且后台继续同步。

### 新接口登录态保活与刷新

- 修复 Apple Account 新接口短效管理态被当成长效状态复用的问题。
- 新接口会按 Apple 返回的管理态 TTL 到期时间刷新 `scnt`、Cookie 和动态 `apiKey`，再使用刷新后的状态创建。
- 新接口创建失败时只保存最后一次成功响应得到的刷新态，非 2xx 失败响应不再覆盖 `scnt` 或 Cookie。
- 新接口浏览器指纹调整为抓包一致的中文语言和东八区时区，减少管理态检测与创建接口的指纹漂移。
- 新接口 401/403 不再一律判定为登录态失效；只有明确返回 authentication/session 失效语义时才要求重新登录，普通 301/404/500/503 会保留原始 HTTP 错误。
- 新接口创建与登录态检测按同一个账号串行执行，避免检测刷新 `scnt` 时创建接口同时使用旧状态。
- 新接口达到隐私邮箱创建上限时会进入短冷却，当前轮回落旧接口或跳过该账号，下一轮再重新尝试。
- 登录前获取 Apple Account 挑战 `scnt` 与登录后管理态保存拆开处理，避免挑战响应污染已保存的新接口登录态。
- 新接口 2FA 通过后会尝试补 Apple 信任确认；失败时不阻断正常保存，只在调试模式输出脱敏摘要。
- 明确说明浏览器 “Remember me” 不等同于旧 iCloud Web 长效 Cookie；旧接口长效来自 `2sv/trust` 和 `extended_login` 换取的 iCloud WebServices Cookie。

### 登录态检测与展示

- 登录态检测同时覆盖 Apple Account 新接口和 iCloud Web 旧接口。
- 新接口登录态检测会复用近期健康状态，避免刚保存的新接口状态被检测刷新误判为异常。
- 每个 Apple 登录态会分别展示“新接口”和“旧接口”的登录状态。
- 前端卡片精简为 Apple ID、账号 ID、新旧接口状态和整体登录态结果。
- Apple ID / 账号邮箱在前端全量显示，不再用星号脱敏。
- 去掉前端卡片里多余的最近检测时间、Cookie 数量、DSID、保存时间等调试信息。
- 新增测试覆盖新接口登录态检测、公开状态字段和新旧接口状态拆分。

### 管理面板 UI

- 管理员后台改为新的分类化数据面板，顶部动作区和数据卡片重新整理。
- 首页提供清透、蓝调、灰阶、舒展、紧凑等前端风格切换组件。
- 账号信息与“账号数据/退出”入口移动到右上角，减少左侧栏拥挤。
- 导出和清理工具从左侧栏移动到“高级工具”分类中，不再常驻挤占导航。
- 日志区域统一展示运行日志、创建账号日志和定时创建日志；切换普通分类时日志保留，切到邮箱池时隐藏。

### 隐私邮箱创建

- 隐私邮箱创建优先使用 Apple Account 新接口。
- 新接口创建前会优先使用近期可用的 `scnt` 和动态 `apiKey`，真正认证失败时再刷新并重试。
- 新接口创建失败后，会继续尝试 iCloud Web 旧接口。
- 新接口与旧接口的创建上限冷却分开记录，新接口达到上限不会把旧接口也判定为冷却中。
- 定时创建按账号并行处理；某个账号本轮失败后，只跳过该账号当前轮次，其他账号继续创建。
- 定时创建只保留定时间隔配置，默认 60 分钟；每轮创建间隔可自定义，默认 5 秒。
- 普通创建和定时创建都支持选择“自动使用新旧接口 / 只用新接口 / 只用旧接口”。
- 创建配置保存到服务器状态文件，刷新页面或换浏览器后仍会保留账号选择、接口策略和定时间隔。
- 定时创建记录已创建数量、失败数量、下一次执行时间和最近错误。
- 定时创建记录支持复制日志和清除日志；清除只清记录，不重置创建/失败计数。
- 创建账号列表里会完整显示 Apple ID，并清楚标记新接口、旧接口和整体登录态状态。
- 创建区域新增“检测登录态”入口，可直接刷新新旧接口状态显示。
- 创建失败会展示账号级失败原因；Apple 新接口错误会带阶段、HTTP 状态码和脱敏后的原始返回摘要，方便判断是接口入口异常、登录态失效、额度上限还是服务端临时错误。
- Apple 新接口 2xx 空响应按临时失败处理，下一轮优先重试新接口，再次失败才回落旧接口继续。
- Apple 新接口遇到 DNS 超时、连接超时、连接重置等底层网络错误时会自动重试 3 次；HTTP 业务错误、额度上限和登录态失效不走网络重试。
- 新接口刷新倒计时到点后会自动调用登录态检测接口，真正刷新 Apple Account 管理态时间，不再只刷新页面状态。
- 登录态检测返回“新接口正常”但未返回刷新倒计时时，会自动继续检测，直到拿到刷新时间或该登录态转为异常。
- 多账号自动刷新改为逐账号判断；某个账号仍显示“待自动刷新”时会继续刷新，不会被其他账号已拿到倒计时而提前停止。
- 新接口刷新倒计时到点后的自动检测节奏调整为每 3 秒一次，避免短时间连续请求。
